# 客户端考试流程说明（前端 / 测试）

本文描述考生在**客户端**完成一次考试时的推荐页面流程、HTTP 接口与状态机，便于前端联调与测试编写用例。实现以当前仓库代码为准；具体 URL 前缀以部署环境为准。

---

## 1. 文档读者与范围

| 读者 | 用途 |
|------|------|
| 前端 | 页面跳转顺序、接口字段、倒计时与交卷边界 |
| 测试 | 用例步骤、状态断言、错误码与配置项 |

**范围**：`/api/client` 下需登录的考试相关接口（见 `internal/cmd/cmd.go` 路由注册）。管理端批次/组卷不在此展开。

---

## 2. 鉴权与基础路径

- **基础路径**：`/api/client`（服务监听端口见 `manifest/config/config.yaml` 中 `server.address`，默认示例为 `:8002`）。
- **考试与「我的考试」**：需在请求头携带学员登录后的 **Bearer Token**（与 `middleware.Auth(UserTypeClient)` 一致）。
- **无需 Token**：`/api/client` 下的登录、验证码等认证接口（与考试接口分组不同）。

统一响应形态以项目既有约定为准（GoFrame `HandlerResponse` + i18n）；业务错误通过 **HTTP 状态码 + body 内 code / message key** 返回。

---

## 3. 总体流程（推荐）

从业务上，一次考试经历：**可查列表 → 拉试卷壳 → 按需拉大题 JSON → 创建答题会话 → 开考 → 答题中保存答案 → 交卷（或超时自动交卷）→ 查看分数相关字段**。

```mermaid
flowchart LR
  A[GET /me/exams] --> B[GET /exam/papers/{paperId}]
  B --> C[GET /exam/papers/{paperId}/sections/{sectionId}]
  C --> D[POST /exam/batches/{batchId}/attempts]
  D --> E[POST /exam/attempts/{id}/start]
  E --> F[PUT /exam/attempts/{id}/answers 可多次]
  F --> G[POST /exam/attempts/{id}/submit]
  E --> H[GET /exam/attempts/{id} 轮询或关键节点]
  H --> F
  H --> G
```

说明：

- **列表**按「批次 + 绑定的 Mock 卷」展开；**创建会话**须带路径中的 `batch_id` 以及 body 中的 `mock_examination_paper_id`（见下文「创建会话」）。
- **试卷初始化**与**大题内容**分两个接口，便于按需加载听力/阅读等大块 JSON。
- **开考**后才会写入 `started_at`、`deadline_at`，此前会话为「未开始」。

---

## 4. 答题会话状态 `status`

定义见 `internal/consts/exam_attempt.go`：

| 值 | 常量含义 | 说明 |
|----|----------|------|
| 1 | 未开始 | 已创建会话，尚未调用开考 |
| 2 | 进行中 | 已开考，可保存答案；超时后服务端可自动视为交卷 |
| 3 | 已交卷 | 用户主动交卷或系统自动交卷后的中间态，随后进入算分收尾 |
| 4 | 已结束 | 客观题等已计入 `exam_result` 等收尾完成（前端以接口返回为准） |

前端建议：**以 `GET /exam/attempts/{id}` 返回为准**刷新 UI；响应中含 `server_time`、`deadline_at`、`deadline_reached` 等字段辅助展示倒计时与超时提示。

## 5. 接口一览

以下路径均相对于 **`/api/client`**（即完整路径形如 `GET /api/client/me/exams`）。

| 方法 | 路径 | 摘要 |
|------|------|------|
| GET | `/me/exams` | 我的考试：按「批次 + 绑定的 Mock 卷」展开多行；每行含 `batch_id`、本行 `mock_examination_paper_id`、同批全部可选卷 `mock_examination_paper_ids`、`exam_start_at` / `exam_end_at`、`window_status` 等 |
| GET | `/exam/papers/{paperId}` | 考前试卷初始化：结构、时长、资源前缀等（**脱敏**，无标准答案） |
| GET | `/exam/papers/{paperId}/sections/{sectionId}` | 单个大题的 topic JSON（脱敏，与资源文件结构一致） |
| POST | `/exam/batches/{batchId}/attempts` | 按批次与 **Mock 卷**创建答题会话（body 必填 `mock_examination_paper_id`），返回 `attempt_id` |
| POST | `/exam/attempts/{id}/start` | 开考；可选 body：`duration_seconds` |
| GET | `/exam/attempts/{id}` | 会话详情：状态、时间、分数字段、服务端时间等 |
| PUT | `/exam/attempts/{id}/answers` | 批量保存答案（可多次） |
| POST | `/exam/attempts/{id}/submit` | 主动交卷 |
| POST | `/exam/papers/{paperId}/attempts/{attemptId}/random-answers` | **仅测试**：随机填答（受配置开关约束） |

听力 HLS：存在仅挂 `Trace`/`Response` 的媒体路由 `GET /api/client/exam/media/hls/{ticket}.m3u8`（**不按考试接口组鉴权**），具体签发与 topic 内字段以前端与当前实现为准。

## 6. 推荐调用顺序（详述）

### 6.1 我的考试

- **GET** `/me/exams`  
- 展示列表；用户选择某场后应拿到 **`batch_id`** 与 **`mock_examination_paper_id`**（该用户在当前行的绑定卷）。若同批次配置了多张卷，列表会出现多行；`mock_examination_paper_ids` 为该批次全部可选卷 id。

### 6.2 试卷壳（初始化）

- **GET** `/exam/papers/{paperId}`，`paperId` = `mock_examination_paper_id`  
- 用于卷面结构、各部分 `section` 的 `id`、`topic_items` / 排序等，**不含**完整题目树（大题详情见下一节）。

### 6.3 大题内容（按需）

- **GET** `/exam/papers/{paperId}/sections/{sectionId}`  
- 返回与 `pt.json` 等一致的 **map JSON**（脱敏）。前端按 `items` 里各 `section` 的 id 多次请求即可。

### 6.4 创建会话

- **POST** `/exam/batches/{batchId}/attempts`  
- Body 可为空（当前实现按用户在 **`exam_batch_member`** 中的一条绑定取卷：`batch_id` + `member_id` + **`exam_paper_id`**）。  
- 前置条件要点（逻辑在 `CreateAttemptForBatch`）：  
  - 批次存在且在 `exam_start_at`～`exam_end_at` 窗口内；  
  - 对应 **`exam_paper`** 已在批次的 **`exam_batch_paper`** 中配置；  
  - 用户已在该批次绑定该 **`exam_paper`**；  
  - 该用户在该批次该 **`exam_paper`** 下**尚无**未删除的 `exam_attempt`（否则 `11121`）。

### 6.5 开考

- **POST** `/exam/attempts/{id}/start`  
- Body 可带 `duration_seconds`（可选）。时长规则见下一节。  
- 成功后 `status` 变为进行中，`deadline_at = started_at + duration`。

### 6.6 保存答案

- **PUT** `/exam/attempts/{id}/answers`  
- Body：`{ "items": [ { "question_id", "answer" }, ... ] }`  
- **客观题**：`answer` 传选项 id（数字即可，服务端会规范为内部 JSON）。  
- **写作等主观题**：`answer` 传字符串文本。  
- 仅 **`status = 进行中`** 且未超过 `deadline_at` 时可成功保存；超时返回业务错误（见错误码表）。  
- `items` 中所有 `question_id` 必须属于本会话绑定的 `exam_paper`，否则参数错误。

### 6.7 交卷

- **POST** `/exam/attempts/{id}/submit`  
- 重复提交已交卷会话：接口可能表现为成功幂等（逻辑上会走收尾）。  
- **超时**：到达 `deadline_at` 后，在用户调用 **GET 详情 / 保存答案** 等路径时，服务端可能触发 **自动交卷并计分**（`maybeAutoSubmitIfOverdue`）。前端仍应主动调 submit 或在 UI 上根据 `deadline_reached` 提示用户。

### 6.8 结果展示

- 交卷并收尾后，**GET** `/exam/attempts/{id}` 中的 `objective_score`、`subjective_score`、`total_score`、`has_subjective` 等供展示（具体含义以前端产品为准）。

## 7. 考试时长与配置

配置项在 **`manifest/config/config.yaml`** 的 `exam` 段（可被环境/配置中心覆盖）：

| 配置项 | 含义 |
|--------|------|
| `defaultDurationSeconds` | 试卷 `duration_seconds` 为 0 时使用的默认考试时长（秒） |
| `maxDurationSeconds` | 开考时客户端传入 `duration_seconds` 的上限（会被夹紧） |
| `saveAnswersPerSecond` | 保存答案限流（每秒） |
| `enableRandomAnswerHelper` | 是否开放随机填答测试接口（**生产应为 false**） |

**ResolveDurationSeconds** 规则简述：

1. 以试卷自带时长为准（若 `> 0`）；否则用 `defaultDurationSeconds`。  
2. 若开考请求带了 `duration_seconds > 0`，则改用客户端值。  
3. 结果不得超过 `maxDurationSeconds`。  

前端应用 **`deadline_at` 与服务端 `server_time`** 计算剩余时间，避免仅依赖本机时钟。

## 8. 测试专用：随机填答

- **POST** `/exam/papers/{paperId}/attempts/{attemptId}/random-answers`  
- 仅当 **`exam.enableRandomAnswerHelper = true`** 时可用；否则返回 **`11005`**（`err.exam_test_helper_disabled`）。  
- 用于自动化快速铺答案；响应中含可拷贝到 **PUT …/answers** 的 `submit_json` 结构（以实际响应为准）。

## 9. 常见业务错误码（考试相关）

错误体中的 `message` 多为 i18n key，下表为 **code** 与 key 对照（节选，便于测试断言）：

| Code | i18n key（示例） | 典型场景 |
|------|------------------|----------|
| 10005 | err.token_required | 未带 Token 访问考试接口 |
| 10010 | err.invalid_params | 参数不合法、题目不属于该试卷等 |
| 11001 | err.exam_attempt_not_found | 会话不存在或不属于当前用户 |
| 11002 | err.exam_time_expired | 已过截止时间仍保存答案 |
| 11003 | err.exam_already_submitted | 非进行中时保存答案等 |
| 11005 | err.exam_test_helper_disabled | 随机填答关闭 |
| 11115 | err.exam_batch_not_found | 批次不存在 |
| 11122 | err.exam_batch_paper_not_in_batch | 该 Mock 卷未纳入批次配置 |
| 11119 | err.exam_batch_window_not_open | 不在批次考试时间窗口内 |
| 11120 | err.exam_batch_member_not_found | 用户未绑定该批次下的该 Mock 卷 |
| 11121 | err.exam_attempt_exists_for_batch | 该批次该 Mock 卷已有会话，不能重复创建 |
| 11123 | err.exam_batch_paper_has_members | 管理端：批次编辑时移除仍绑定学员的卷 |

完整列表见 `internal/consts/errors.go`。

## 10. 测试检查清单（建议）

1. **窗口外**：批次未到开始或已结束，创建会话失败 `11119`。  
2. **未报名**：创建会话失败 `11120`。  
3. **重复会话**：同一用户同批次同 Mock 卷第二次创建失败 `11121`。  
4. **状态机**：未开始 → 开考 → 进行中；未开考不可保存答案。  
5. **倒计时**：开考后 `deadline_at` 与 `server_time` 一致；超时后保存失败 `11002`，随后 GET 可能已变为已交卷/已结束。  
6. **主动交卷**：submit 后再次保存应失败（`11003` 等）。  
7. **试卷加载**：先 GET paper，再按 section 拉 topic，字段与听力资源 URL 拼接正常。  
8. **限流**：高频 PUT answers 是否触发限流（依 `saveAnswersPerSecond`）。  
9. **随机填答**：仅在 `enableRandomAnswerHelper=true` 环境启用自动化用例。
